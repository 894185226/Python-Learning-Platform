# 初中信息科技 · Python 变量 交互式教学网站规划方案

## 整体设计思路
- **生活化类比先行**：以“贴标签的盒子”贯穿始终，降低抽象概念的理解门槛。
- **即时反馈全程伴随**：每一步操作都有对错提示与鼓励，保护学习信心。
- **低地板，高天花板**：简单任务可拖拽完成，挑战任务可触及变量交换、多变量操作。
- **完整学习闭环**：从情境引入到成就结算，形成清晰的仪式感与学习路径。

## 功能模块规划表

| 模块名称 | 功能描述 | 教学目标 |
|---|---|---|
| **1. 情境导入模块**<br>（故事/动画） | 通过动画展示“记不住的数字”问题：主角需要记录变化的分数、名字或倒计时，引出“会变的小盒子”。学生点击不同场景，看到数值在不断变化。 | 感知变量是用来存储变化数据的；激发“我也要做一个”的学习动机。 |
| **2. 生活类比实验室**<br>（拖拽互动） | 屏幕左侧为“变量盒子”，右侧为“标签”与“数据物品”。学生拖拽标签到盒子上，再拖入数据，盒子外观随类型变化（数字蓝，文字绿）并显示当前值。可多次更换盒内物品。 | 形象理解变量名（标签）、变量值（盒中物）、数据类型；直观感受赋值就是“把东西放进去”。 |
| **3. 知识讲解模块**<br>（结构化微课） | 2~3分钟短视频配合图文卡片，讲解变量定义、命名规则（字母/数字/下划线、不能数字开头、区分大小写）、赋值符号 `=` 的含义。卡片可点击展开更多示例。 | 明确变量核心概念与命名规范；能区分合法与非法的变量名；理解赋值不是数学中的相等。 |
| **4. 命名小法官**<br>（闯关判断） | 以趣味“法庭审判”形式，展示变量名（如 `2name`、`my_name`、`class`、`年龄`），学生点击“合法✅”或“非法❌”，即时给出对错动画与规则解释。连对可积星。 | 强化命名规则，通过正反例辨析，在实际编程中能避免常见命名错误。 |
| **5. 实践操作模块**<br>（分级在线编程） | 分屏设计：左为指令提示区，右为代码编辑器与运行窗口。Lv1：补全赋值语句并运行；Lv2：独立写出赋值与打印；Lv3：修改变量值，观察输出变化。 | 能独立完成变量的声明、赋值与输出；体验“运行-观察-修改”的编程循环。 |
| **6. 值追踪挑战**<br>（脑力追踪） | 展示多行代码（如 `x=5; y=x+2; x=10`），学生需在每一步后指出变量的当前值。通过高亮动画逐步执行，学生拖拽数字到变量旁，系统判断对错。 | 理解变量值的“覆盖”特性，建立程序顺序执行的思维模型，为多变量操作打基础。 |
| **7. 错误调试诊所**<br>（找茬修复） | 展示含有bug的代码片段（未定义就使用、引号缺失、命名非法等），学生点击错误行，并从选项中选出正确修复方案。修复成功获得“修复勋章”。 | 培养阅读与调试简单错误的能力，建立“报错不可怕，我能修好”的信心。 |
| **8. 扩展思维模块**<br>（拔高挑战） | 包含两个挑战：① 变量交换（两杯水如何互换，引入第三杯）；② 用变量拼接句子（姓名、年龄、爱好组合输出）。配有可视化动画辅助理解。 | 思维进阶：理解交换需借助临时变量；能综合运用多变量解决简单实际问题。 |
| **9. 创意迷你项目**<br>（学以致用） | 提供“自我介绍生成器”半成品模板。学生修改变量 `name`、`age`、`hobby` 的值，点击运行即可生成带插图的电子名片或对话气泡。 | 在真实小项目中体验变量的用途，提升成就感，连接编程与自我表达。 |
| **10. 课堂小测 & 能力星图**<br>（评估反馈） | 5道选择题/拖拽排序题，覆盖核心概念。完成后自动生成雷达图，显示变量定义、命名、赋值、调试、应用五个维度的掌握程度（1~3星）。 | 检测学习效果，可视化定位薄弱点，为后续学习提供参考。 |
| **11. 回顾与成就墙**<br>（总结仪式） | 以“变量岛探险地图”的形式串联所有模块，已完成模块亮起，汇总展示获得的星星与勋章。点击可复习核心口诀（如“变量是个盒，标签我来贴，数据放进去，用谁拿谁”）。 | 形成完整学习闭环，强化记忆，积累成就感，保持持续学习动力。 |
