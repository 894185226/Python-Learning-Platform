# 提示词：Python 基础入门 · 交互式专题学习网站（模仿「变量」模块模式扩展）

## 一、项目目标
请帮我创建一个完整的 **Python 基础入门交互式学习网站**。该网站将现有的「变量」专题学习模块作为其中一个章节，并按照完全相同的交互模式和教学风格，扩展出覆盖 Python 核心基础知识的全部章节。整个网站需面向初一、初二学生，基础薄弱，因此所有内容都必须通过生活化类比、拖拽互动、即时反馈、分级挑战、调试诊所、创意项目等方式来呈现，保护学生的学习信心和成就感。

## 二、整体要求
1. **保留现有「变量」模块的全部内容**：包括情境导入、生活类比实验室、知识讲解、命名小法官、实践操作、值追踪挑战、错误调试诊所、扩展思维、创意迷你项目、课堂小测、回顾成就墙这 11 个交互子模块。不允许删减或改变其交互方式。
2. **其余所有新章节**，必须严格模仿「变量」模块的模式，每个章节至少包含以下核心子模块（可根据内容适当增减，但必须保持教学闭环）：
   - **情境导入**（动画/故事，引出问题）
   - **知识讲解**（短微课视频卡片或图文卡片）
   - **互动实验室**（拖拽、模拟、可视化理解核心概念）
   - **实践操作**（分级编程挑战，补全→独立写→修改观察）
   - **闯关/判断**（正误判断或拖拽排序，即时反馈）
   - **调试诊所**（找出代码错误并修复，获得勋章）
   - **扩展思维/挑战**（拔高任务，如变量交换、图形打印等）
   - **创意小项目**（半成品模板，修改代码生成作品）

   - **成就回顾**（章节地图，星星汇总）
3. 网站采用单页应用 (SPA) 或清晰的侧边栏导航，可按章节顺序学习，进度自动保存（可用 localStorage）。
4. 界面风格统一：明亮友好，卡通化图标，成功绿色动画，错误橙色鼓励弹窗（无红色叉号），全程有漂浮小助手可点击重复提示。
5. 所有编程交互必须内嵌一个简单的 Python 代码编辑器（或使用 Brython/Skulpt 等前端执行，或模拟执行结果），对于无法真实运行的步骤，使用预设模拟反馈。
