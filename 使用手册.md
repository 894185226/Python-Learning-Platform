## 零、写在前面（请一定先读完这一段！）

本手册会**手把手**教你如何在任何一台 Windows 电脑上把这个网站跑起来，**完全不需要任何编程基础**。

整个流程概括为一句话：

> **把文件夹复制到新电脑 → 双击一个文件 → 等待自动完成 → 浏览器自动打开网站**

是的，就这么简单。如果中途遇到任何问题，请翻到**第九节「常见问题」**，那里有 15 个常见问题的详细解答。

**重要提示**：
- 本手册中所有"双击"都是指用鼠标左键快速点击两下
- 如果你看到不认识的英文单词或代码，不要担心，你不需要理解它们
- 所有操作都在 Windows 系统上进行，Mac 电脑暂不支持自动部署

## 一、项目简介

本项目是一个面向中学生的 **Python 基础互动学习平台**，覆盖从 Python 入门到进阶的完整知识体系。

### 1.1 技术架构一览

| 技术层次 | 技术选型 | 说明 |
|----------|----------|------|
| 前端 | HTML5 + CSS3 + JavaScript | 学生看到的网页界面 |
| 后端 | Node.js + Express | 处理数据请求的服务器程序 |
| 数据库 | MySQL | 存储学生信息、学习进度等数据 |
| 启动方式 | 双击 `start_server.bat` | 自动调用 PowerShell 脚本完成启动 |

### 1.2 网站内容规模

- **19 个章节**，共 **139 个学习模块**
- 每个章节包含：情境导入、知识讲解、互动实验室、实践操作、调试诊所、扩展思维、创意项目、课堂小测等完整教学环节
- **24 项成就系统**（19 个章节成就 + 4 个里程碑成就 + 1 个终极成就「全能学霸」）
- 通过闯关激励机制提升学生学习兴趣

### 1.3 管理平台功能

提供完整的教学管理功能：
- 学生账号管理（增删改查、批量导入/转班/毕业/重置密码）
- 数据统计仪表盘（学习进度、活跃度、登录记录）
- 公告发布、作业管理、每日一题
- 操作日志记录

## 三、快速开始（两种方式，选一种即可）

### 方式一：日常启动（推荐，每次上课用）

这是你**最常用**的方式。每次上课前，在教师机上操作：

> **双击 `start_server.bat`**

这个文件只有 3 行代码，它会自动调用 `start_server.ps1` 脚本来完成以下工作：

1. ✅ 检查 Node.js 是否已安装
2. ✅ 检查 MySQL 是否在运行（如未运行则自动尝试启动）
3. ✅ 释放被占用的 3000 端口
4. ✅ 检测教师机的局域网 IP 地址
5. ✅ 配置 Windows 防火墙（自动放行 3000 端口，让学生机能访问）
6. ✅ 自动打开浏览器
7. ✅ 启动网站服务器

**启动成功后**，命令行窗口会显示类似这样的信息：

```
==============================================
  Python Variable Adventure
==============================================

  Teacher (this machine):
    http://localhost:3000
    http://localhost:3000/admin.html

  Students (LAN - share this address):
    http://192.168.0.104:3000

  Press Ctrl+C to stop the server
==============================================
```

**访问地址说明**：

| 地址 | 用途 | 谁用 |
|------|------|------|
| `http://localhost:3000` | 网站主页 | 教师自己在本机访问 |
| `http://localhost:3000/admin.html` | 管理后台 | 教师管理学生和数据 |
| 局域网地址（如 `http://192.168.0.104:3000`） | 学生访问地址 | 学生在自己电脑浏览器中输入 |

**管理后台默认账号**：
- 用户名：`admin`
- 密码：`admin123`

> **关闭网站**：在命令行窗口按 `Ctrl + C`，然后关闭窗口即可。

### 方式二：首次部署（新电脑第一次用）

当你把网站文件夹复制到一台**全新的电脑**上时，使用这个方式。

> **双击 `setup_server.bat`**

这个文件会自动调用 `setup_server.ps1` 脚本来完成全部环境搭建：

| 阶段 | 做什么 | 大概需要多久 |
|------|--------|-------------|
| Phase 1 | 检查/安装 Node.js | 如果没装，2-5 分钟 |
| Phase 2 | 安装 npm 依赖包 | 1-2 分钟 |
| Phase 3 | 检查/安装 MySQL | 如果没装，3-5 分钟 |
| Phase 4 | 配置 MySQL（初始化数据目录、创建服务、启动） | 30 秒 - 2 分钟 |
| Phase 5 | 初始化数据库（创建库和表） | 5 秒 |
| Phase 6 | 启动网站 | 即时 |

> **注意**：首次运行时会弹出 UAC（用户账户控制）窗口（一个蓝色背景的对话框），请点击"是"以授予管理员权限。这是正常的，因为脚本需要配置防火墙和 MySQL 服务。

**全部完成后**，网站会自动在浏览器中打开。之后每次使用只需双击 `start_server.bat` 即可。

## 四、电脑室局域网部署（重要！）

如果你要在电脑室让学生通过局域网访问网站，请按以下步骤操作。

### 4.1 教师机操作（每次上课前）

1. 在教师机上双击 `start_server.bat`
2. 等待命令行窗口显示启动信息（约 5-10 秒）
3. 在窗口中找「Students (LAN)」后面的地址，例如：`http://10.20.30.15:3000`
4. 将这个地址告诉学生（写在黑板上或通过电子教室软件广播）

### 4.2 学生机操作（学生第一次使用时）

1. 打开浏览器（推荐 Chrome 或 Edge）
2. 在地址栏输入教师机显示的局域网地址，例如：`http://10.20.30.15:3000`
3. 按回车键，即可看到网站首页
4. 点击页面右上角的「注册」按钮
5. 填写注册信息：
   - **姓名**：你的真实姓名（如：张三）
   - **用户名**：登录用的账号（如：zhangsan，建议用姓名拼音）
   - **密码**：自己记住的密码（至少 6 位）
   - **年级**：选择你的年级
   - **班级**：输入你的班级（如：1班）
6. 点击「注册」按钮完成注册
7. 注册成功后，用刚才的用户名和密码登录即可开始学习

### 4.3 学生机操作（之后每次上课）

1. 打开浏览器
2. 输入局域网地址
3. 输入用户名和密码，点击登录
4. 从上次的学习进度继续

### 4.4 注意事项

- 教师机和学生机必须在**同一个局域网**内（例如连接同一个交换机）
- 教师机的防火墙需要放行 3000 端口（脚本已自动配置，无需手动操作）
- **教师机不能关闭命令行窗口**，否则网站就停了，所有学生都会掉线
- 如果学生无法访问，首先检查：
  1. 教师机命令行窗口是否还开着（那个黑色窗口）
  2. 学生输入的地址是否正确（注意是 `http://` 不是 `https://`）
  3. 教师机 IP 地址是否发生了变化

### 4.5 电脑室固定 IP 段说明

如果你的电脑室 IP 段是 `10.20.30.XX`，脚本会自动检测教师机的具体 IP（例如 `10.20.30.15`）并显示在启动信息中。学生端只需输入对应的地址即可。

## 六、管理平台详解（教师端）

访问地址：**http://localhost:3000/admin.html**

默认账号：**admin**
默认密码：**admin123**

> **安全提示**：首次登录后，强烈建议立即修改默认密码！修改方法见第九节 Q7。

### 6.1 登录管理后台

1. 在教师机浏览器中输入 `http://localhost:3000/admin.html`
2. 输入用户名 `admin` 和密码 `admin123`
3. 点击「登录」按钮
4. 登录成功后，你会看到左侧深色导航栏和右侧数据概览页面

### 6.2 数据概览（仪表盘）

登录后默认显示的页面，包含：

- **四个统计卡片**：注册学生数、模块完成数、成就获得数、登录次数
- **模块完成统计图**：各模块的完成人数分布
- **最近登录记录**：显示最近登录的学生和时间
- **学习趋势图**：近 30 天的学习活跃度变化
- **章节完成率**：每个章节的完成比例
- **测验成绩汇总**：各章节的平均分和参与人数

### 6.3 学生管理

点击左侧「学生管理」进入，可以：

- **搜索学生**：输入姓名或用户名进行搜索
- **筛选学生**：按年级、班级筛选
- **查看详情**：点击学生姓名查看详细学习数据（模块进度、成就、登录日志）
- **批量操作**：
  - 转班：将选中的学生转到其他班级

## 七、数据备份与恢复（重要！）

定期备份数据可以防止意外丢失学生的学习记录。

### 7.1 备份数据库

**方法一：使用命令行备份（推荐）**

1. 打开命令提示符（按 `Win + R`，输入 `cmd`，回车）
2. 输入以下命令（将 `保存路径` 替换为你想要保存的位置）：

```
mysqldump -u root python_var_lesson > "保存路径\backup_%date:~0,4%%date:~5,2%%date:~8,2%.sql"
```

例如备份到桌面：
```
mysqldump -u root python_var_lesson > "C:\Users\你的用户名\Desktop\backup.sql"
```

**方法二：直接复制数据库文件夹**

1. 停止 MySQL 服务（按 `Win + R`，输入 `services.msc`，找到 MySQL 服务，右键停止）
2. 复制 `C:\ProgramData\MySQL\MySQL Server X.X\Data\python_var_lesson\` 整个文件夹到备份位置
3. 重新启动 MySQL 服务

### 7.2 恢复数据库

如果需要恢复备份的数据：

```
mysql -u root python_var_lesson < "备份文件路径.sql"
```

### 7.3 备份建议

- 建议**每周至少备份一次**
- 学期末务必做一次完整备份
- 备份文件可以保存在 U 盘或云盘上
- 备份文件名建议包含日期，方便识别

## 八、跨电脑移植（把网站搬到另一台电脑）

### 8.1 完整移植步骤

1. 将整个项目文件夹（包含所有文件和子文件夹）复制到 U 盘或通过网络传输
2. 在新电脑上粘贴到任意位置（例如桌面，**建议路径不要包含中文**）
3. 双击 `setup_server.bat`（首次部署）
4. 等待脚本自动完成所有配置（约 5-15 分钟，取决于网速和电脑性能）
5. 之后每次使用只需双击 `start_server.bat` 即可

### 8.2 移植时保留学生数据

如果你想把旧电脑上的学生数据也搬到新电脑：

1. 在旧电脑上，按照第七节的方法备份数据库
2. 将备份的 `.sql` 文件复制到新电脑
3. 在新电脑上完成 `setup_server.bat` 部署后
4. 打开命令提示符，执行：
   ```
   mysql -u root python_var_lesson < "备份文件路径.sql"
   ```
5. 数据就全部迁移到新电脑了

### 8.3 注意事项

- 确保目标电脑能联网（首次启动需要下载 Node.js 和 npm 依赖）
- 首次启动时会弹出 UAC 窗口，请点击"是"
- 如果两台电脑使用不同的 MySQL root 密码，脚本会自动处理
- 移植后数据库是全新的，需要手动导入备份数据才有学生记录

## 九、常见问题（小白必看！）

### Q1：双击 .bat 文件后窗口闪退？

**原因**：通常是因为环境问题或文件路径问题。

**解决方法**：
1. 右键点击 `start_server.bat`，选择「以管理员身份运行」
2. 如果还是闪退，说明你的电脑缺少 Node.js，请双击 `setup_server.bat` 进行首次部署
3. 确保项目文件夹路径中没有中文或特殊字符

### Q2：双击后窗口显示乱码？

**原因**：这是 Windows 命令行的编码问题，**不影响实际功能**。

**解决方法**：直接忽略，只要看到 `http://localhost:3000` 这样的地址就说明成功了。

### Q3：打开网站后无法登录，显示"服务器错误"？

**解决方法**：
1. 确认命令行窗口（黑色窗口）没有关闭，服务器正在运行中
2. 确认 MySQL 服务已启动：
   - 按 `Win + R`，输入 `services.msc`，回车
   - 在服务列表中找到 `MySQL` 或 `MySQL80`
   - 如果状态不是"正在运行"，右键选择"启动"
3. 重新双击 `start_server.bat`

### Q4：学生无法通过局域网访问我的网站？

**解决方法**（按顺序检查）：
1. 确认教师机和学生机在同一个局域网（连接在同一个交换机上）
2. 在教师机浏览器中输入 `http://localhost:3000`，确认网站能正常访问
3. 在教师机浏览器中输入局域网地址（如 `http://10.20.30.15:3000`），确认能从外部访问
4. 检查教师机防火墙是否阻止了 3000 端口：
   - 打开「Windows 安全中心」→「防火墙和网络保护」→「高级设置」
   - 在「入站规则」中查找是否有 Node.js 或端口 3000 的规则
   - 如果没有，可以手动添加一条入站规则放行端口 3000
5. 如果以上都正常，让学生尝试用 Chrome 浏览器访问

### Q5：端口 3000 被占用怎么办？

**原因**：上次没有正常关闭服务器，或者有其他程序在使用 3000 端口。

**解决方法**：脚本已自动处理端口占用问题。如果仍然失败，手动执行：
1. 按 `Win + R`，输入 `cmd`，回车
2. 输入 `netstat -ano | findstr :3000`
3. 记下最后一列的数字（PID，例如 12345）
4. 输入 `taskkill /f /pid 12345`（把 12345 换成你记下的数字）
5. 重新双击 `start_server.bat`

### Q6：MySQL 安装后一直连不上？

**解决方法**：`setup_server.ps1` 脚本会自动尝试以下操作：
1. 通过 Windows 服务启动 MySQL
2. 直接启动 mysqld 进程
3. 通过安全模式重置 root 密码

如果以上全部失败，请手动检查：
- 按 `Win + R`，输入 `services.msc`，确认 MySQL 服务存在且可启动
- 确认 MySQL 数据目录已初始化（`C:\ProgramData\MySQL\` 下应有 `Data` 文件夹）
- 尝试手动执行：以管理员身份打开命令提示符，输入 `mysqld --initialize-insecure --console`

### Q7：如何修改管理员密码？

**解决方法**：
1. 用记事本打开 `server.js`
2. 按 `Ctrl + F`（查找），搜索 `admin123`
3. 将 `admin123` 改为你的新密码（注意保留两边的引号）
4. 保存文件
5. 关闭命令行窗口，重新双击 `start_server.bat`

### Q8：如何重置某位学生的密码？

**方法一（推荐）**：在管理后台（`http://localhost:3000/admin.html`）找到该学生，勾选后点击「批量操作」→「重置密码」。

**方法二**：通过 MySQL 命令行执行（以用户名 `test001` 为例）：

```sql
UPDATE python_var_lesson.students SET password = '123456' WHERE username = 'test001';
```

### Q9：如何删除某位学生的所有数据？

在管理后台找到该学生，点击「删除」按钮即可。如需手动操作：

```sql
SET @sid = (SELECT id FROM python_var_lesson.students WHERE username = 'test001');
DELETE FROM python_var_lesson.learning_progress WHERE student_id = @sid;
DELETE FROM python_var_lesson.achievements WHERE student_id = @sid;
DELETE FROM python_var_lesson.login_logs WHERE student_id = @sid;
DELETE FROM python_var_lesson.mistake_book WHERE student_id = @sid;
DELETE FROM python_var_lesson.study_notes WHERE student_id = @sid;
DELETE FROM python_var_lesson.code_snippets WHERE student_id = @sid;
DELETE FROM python_var_lesson.student_goals WHERE student_id = @sid;
DELETE FROM python_var_lesson.notifications WHERE student_id = @sid;
DELETE FROM python_var_lesson.discussion_posts WHERE student_id = @sid;
DELETE FROM python_var_lesson.assignment_submissions WHERE student_id = @sid;
DELETE FROM python_var_lesson.students WHERE id = @sid;
```

### Q10：.ps1 文件是什么？双击打开是记事本？

**回答**：`.ps1` 是 PowerShell 脚本文件。**你不需要直接双击它**，只需要双击 `.bat` 文件即可，`.bat` 会自动调用 `.ps1`。Windows 默认双击 `.ps1` 会用记事本打开，这是安全设计，防止误运行脚本。

### Q11：setup_server.bat 和 start_server.bat 有什么区别？

**回答**：
- `setup_server.bat`：**首次部署用**。会自动安装 Node.js、MySQL、npm 依赖包，初始化数据库。只需要在新电脑上运行一次。
- `start_server.bat`：**日常启动用**。只启动网站服务器，不安装任何东西。每次上课前双击即可。

### Q12：可以同时有多少学生在线使用？

**回答**：网站设计支持 **40-50 名学生**同时在线使用，适合一个班级的电脑室教学场景。连接池配置为 20 个连接，速率限制为 60 次/分钟。

### Q13：学生需要安装什么软件吗？

**回答**：不需要。学生只需要一个浏览器（推荐 Chrome 或 Edge），输入教师机的局域网地址即可访问。不需要安装任何软件或插件。

### Q14：网站支持亮色/暗色主题吗？

**回答**：支持。网站右上角有一个太阳/月亮图标按钮，点击即可在亮色和暗色主题之间切换。亮色主题适合白天使用，暗色主题适合光线较暗的环境或保护视力。

### Q15：如何更新网站内容？

**回答**：如果你修改了 `chapters.js` 中的章节内容或 `index.html` 中的页面结构：
1. 保存修改后的文件
2. 关闭命令行窗口（停止服务器）
3. 重新双击 `start_server.bat` 启动服务器
4. 刷新浏览器页面即可看到更新后的内容

## 十一、技术架构详解

```
┌─────────────────┐        HTTP         ┌─────────────────┐        SQL         ┌─────────────┐
│   浏览器客户端   │  ←──────────────→  │  Node.js 服务器  │  ←──────────────→  │   MySQL     │
│   (Chrome)      │     JSON/REST      │    Express       │                    │   数据库     │
└─────────────────┘                    └─────────────────┘                    └─────────────┘
                                              │
                                       监听 0.0.0.0:3000
                                       支持局域网访问
```

- **前端**：HTML5 + CSS3 + JavaScript（无框架，纯原生开发，零依赖）
- **后端**：Node.js + Express（`server.js`），监听 `0.0.0.0:3000`，支持局域网访问
- **数据库**：MySQL，自动创建数据库 `python_var_lesson` 和全部数据表
- **启动脚本**：
  - `start_server.bat` → `start_server.ps1`（日常启动）
  - `setup_server.bat` → `setup_server.ps1`（首次部署）
- **连接池**：20 个连接，支持 40-50 人同时在线
- **速率限制**：60 次请求/分钟，防止误操作
- **静态缓存**：除 HTML 外，静态文件缓存 1 小时，减少局域网带宽占用
- **密码安全**：使用 bcrypt 加密存储密码，不保存明文
